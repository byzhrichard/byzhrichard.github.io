矩阵压缩技术

主要用于存储大规模稀疏矩阵——即大部分元素为 0 的矩阵

COO（Coordinate Format，坐标格式）

直接记录每个非零元素的：

  • 行索引（row）
  • 列索引（col）
  • 数值（value）

存储结构:

row:   [r1, r2, r3, ...]
col:   [c1, c2, c3, ...]
data:  [v1, v2, v3, ...]

优点：

  • 最简单直观
  • 易于构建（适合“插入式构建”）

缺点：

  • 不适合计算（查找慢）
  • 不支持高效矩阵乘法

矩阵：

0  0  3
4  0  0
0  5  0

COO表示：

row = [0, 1, 2]
col = [2, 0, 1]
data= [3, 4, 5]

CSR（Compressed Sparse Row，压缩行存储）

按“行”压缩存储非零元素

存储结构:

data   = 非零值
indices= 对应列索引
indptr = 每一行的起始位置

优点：

  • 按行访问非常快
  • 非常适合矩阵乘法（SpMV：Sparse Matrix × Vector）
  • 工业/深度学习最常用格式之一

缺点：

  • 插入/修改代价高
  • 按列访问慢

indptr生成方式: (总元素个数=行数+1)

  • 从0开始累加 -> 0作为第一个元素
  • 每行有几个非0元素, 则加几, 得到下一个元素
  • …

-> 这样就可以知道每行的元素都是data里的哪些 -> 也可以很快反推每一行都有几个非零元素

矩阵：

0  0  7  0  0
2  0  0  0  9
0  0  0  0  0
0  3  0  4  0

CSR表示：

data    = [7, 2, 9, 3, 4]
indices = [2, 0, 4, 1, 3]
indptr  = [0, 1, 3, 3, 5]
  • 第0行 -> data[0:1] = [7]
  • 第1行 -> data[1:3] = [2, 9]
  • 第2行 -> data[3:3] = [0]
  • 第3行 -> data[3:5] = [3, 4]

CSC（Compressed Sparse Column，压缩列存储）

和 CSR 完全对称：按“列”压缩

存储结构:

data   = 非零值
indices= 对应行索引
indptr = 每一列起始位置

优点：

  • 按列访问快
  • 适合转置操作、列计算
  • 常用于线性代数库（如 MATLAB / SciPy）

缺点：

  • 行访问慢
  • 插入代价高

矩阵：

0  0  3
4  0  0
0  5  0

CSC表示：

data    = [4, 5, 3]
indices = [1, 2, 0]
indptr  = [0, 1, 2, 3]

总结

格式 核心思想 优点 缺点 最适合
COO (row, col, value) 最简单、易构建 计算慢 构建阶段
CSR 按行压缩 行操作快、SpMV快 列操作慢 深度学习/图计算
CSC 按列压缩 列操作快 行操作慢 数值计算/线代
  • COO：我只记录“坐标 + 值”
  • CSR：我按“行”把数据压扁
  • CSC：我按“列”把数据压扁
