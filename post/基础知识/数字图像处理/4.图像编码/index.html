图像编码

霍夫曼编码

霍夫曼编码使用变长编码表对源符号（如文件中的一个字母）进行编码，出现机率的字母使用较短的编码，反之出现机率低的则使用较长的编码，这便使编码之后的字符串的平均长度降低，从而达到无损压缩数据的目的

过程略

概率大, 编码也越短 概率小, 编码也越长

局限性:

  • 源符号集的概率分布不是2负n次方的形式, 则无法达到熵极限
  • 码复杂, 需要知道输入符号集的概率分布, 没有错误保护功能

LZW编码

LZW编码就是把出现过的字符串映射到记号上，这样就可能用较短的编码来表示长的字符串，实现压缩

1. 编码算法

编码器从原字符串不断地读入新的字符，并试图将单个字符或字符串编码为记号

维护两个变量:

  • P (Previous): 已经存在字典里、能对应一个编码数字，但还没输出的一串字符 (相当于：手里攥着一段已经认识的字符串，先不发出去，看看后面能不能拼更长的串)
  • C (Current): 刚从原文里读进来的单个新字符

初始规则: 一开始字典只存所有单独基础字符，每个字符对应一个数字记号

步骤:

  1. 字典只放单个基础字符；P、C 全部为空
  2. 读入新字符 C，拼接成 P+C
    • 还有字符没读完：读出下一个字符 C，把手里的 P 和新 C 拼在一起，得到新串P+C
    • 没有字符可读(到文末)：直接把手里 P 对应的数字输出，算法结束
  3. 判断拼接后的串 P+C 在不在字典里
    • 字典里能找到 P+C -> 把手里攥的 P 更新成 P+C
    • 字典里找不到 P+C
      1. 输出当前手里 P 对应的数字
      2. 给新串P+C分配一个全新数字，存入字典
      3. 重置手里的 P = C，现在手里只攥刚读进来的单个字符 C
  4. 循环回到步骤 2，继续读下一个字符，直到全文读完

2. 解码算法

维护两个变量:

  • pW (Previous Word): 上一个读完的记号（数字）
  • cW (Current Word): 现在刚读进来的新记号（数字）
  • Str(记号)：字典里这个数字对应的原始字符串

步骤:

  1. 初始化：字典只有 0→a、1→b、2→c；pW、cW 空
  2. 读第一个记号 cW，直接输出 Str(cW)（第一个一定是基础字符）
  3. 把 pW = cW（把当前记号存为 “上一个记号”）
  4. 读下一个新记号 cW
  5. 判断 cW 在不在字典
    • cW 在字典里（绝大多数普通情况）
      1. 输出 Str(cW)
      2. 取出：P = Str(pW)（上一个记号对应的字符串）、C = Str(cW) 的第一个字符
      3. 拼接 P+C，分配下一个新数字，加入字典
    • cW 不在字典里（特殊异常情况，叫 LZW 未定义码 / 单例码）
      1. 取出：P = Str(pW)、C = Str(pW) 的第一个字符
      2. 拼接 P+C，这个新串的记号就是当前 cW，存入字典
      3. 输出 P+C
      4. 回到步骤 3 循环，直到所有数字记号读完
