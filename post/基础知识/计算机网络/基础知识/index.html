IP

机器的门牌号: IP 地址

192.168.1.50

1. IPv4

IPv4 是现在最经典、最常见的 IP 地址格式 由 4 段数字组成，每段范围是 0 ~ 255

  • 192.168.1.10
  • 8.8.8.8
  • 114.114.114.114

IPv4 总共是 32 位，所以理论地址数量大概是：2^32 ≈ 43 亿个

听起来很多，但现在手机、电脑、服务器、摄像头、物联网设备太多了，IPv4 地址早就不够用了

内网 IP:

  • 192.168.1.10
  • 192.168.0.5
  • 10.0.0.12
  • 172.16.5.20

公网 IP: (路由器对外可能有一个公网出口 IP，或者运营商给你分配了一个大内网出口)

你的电脑 192.168.1.20
    ↓
路由器 NAT
    ↓
公网出口 IP
    ↓
目标网站

IPv4 时代特别重要的东西：NAT

2. IPv6

IPv6 是为了解决 IPv4 地址不够而设计的新一代 IP 协议

  • 2408:8207:78cc:1234:abcd:ef01:2345:6789

IPv6 是 128 位地址，地址数量极其巨大 -> 以后每台设备都可以有自己的公网地址

IPv6 理论上不再依赖 NAT，每台设备都可以直接拥有全球唯一地址

IPv6 问题:

不是 IPv6 本身不好，而是你的网络处于一种尴尬状态:

  • 系统以为 IPv6 可用
  • 软件优先尝试 IPv6
  • 但实际 IPv6 路径不稳定、不可达、速度慢或者 DNS 异常

不是完全没有 IPv6，而是“看起来有，实际不好用”

有些软件会很聪明: IPv6 不行，我马上回退 IPv4 但有些软件不够聪明: 如steam下载cs2

3. 本机地址 127.0.0.1

  • 127.0.0.1 = 我自己这台电脑
  • localhost = 通常也是我自己这台电脑

只允许服务器自己访问这个服务: app.run(host="127.0.0.1", port=5000)

4. 所有网卡地址 0.0.0.0

  • 0.0.0.0 = 希望内网其他机器访问，要监听

监听这台机器上的所有网卡地址: app.run(host="0.0.0.0", port=5000)

域名

一般不喜欢记 IP，所以会用名字 这些名字不是机器真正的地址

真正通信前，需要先翻译成 IP -> 这个“翻译工作”就是 DNS 做的

server.xxx.com → 192.168.1.50

DNS

DNS = 网络世界的通讯录

所以访问一个域名时，其实分两步：

  • 第一步：查 DNS，拿到 IP
  • 第二步：根据 IP 去连接目标机器

电脑为了提高速度，不会每次都重新问 DNS，它会缓存结果 但问题来了，如果服务器 IP 变了呢

ipconfig /flushdns

这时候刷新dns就是把本机记住的 DNS 翻译结果清空，重新查一遍

重新查后得到新 IP, 于是又能连上了

代理

Clash = 本地代理 + 分流规则 + DNS 处理 + 流量转发

不开 Clash 时:

你的电脑 → 运营商 → 目标网站

开 Clash 时:

你的电脑 → Clash 本地端口 → 代理节点 → 目标网站

代理改变了至少三件事：

  • 访问路径变了
  • 出口 IP 变了
  • DNS 解析方式可能也变了

1. 系统代理

Clash 一般会在本机开一个代理端口，比如：127.0.0.1:7890

浏览器访问网站时，如果它遵守系统代理设置，它会这样走：

浏览器 → 127.0.0.1:7890 → Clash → 代理节点 → 目标网站

2. Tun模式

但问题是：不是所有软件都听系统代理 有些软件会自己联网, 直接: 软件 → 网卡 → 路由器 → 互联网

TUN 模式不太依赖软件配合，因为它在更底层接管流量 TUN 模式会在系统里创建一个 虚拟网卡

原来：

软件
  ↓
真实网卡
  ↓
路由器
  ↓
互联网

开启 TUN 后：

软件
  ↓
Clash 虚拟网卡 TUN
  ↓
Clash 规则判断
  ↓
代理 / 直连 / 拦截
  ↓
真实网卡
  ↓
互联网

所以 TUN 模式不是让软件主动去找 Clash，而是 Clash 站在网络出口前面，把流量先接过来，再决定怎么处理

网关

网关: 这台电脑出门上网的出口 在家里，一般路由器就是网关

比如你的电脑 IP 是：192.168.1.20

网关可能是：192.168.1.1

意思是：如果你要访问外面的网络，就把数据交给 192.168.1.1

电脑 → 网关 → 外网

当你连手机热点时，电脑的网关可能不再是家里路由器，而是手机：

电脑 → 手机 → 家里路由器 → 外网

所以即使最终出口还是同一个 WiFi，中间网络结构已经不同了

NAT

NAT: 多个设备共用一个外网地址上网

比如你家有三台设备：

  • 电脑：192.168.1.20
  • 手机：192.168.1.21
  • 平板：192.168.1.22

这些都是内网地址，外网网站一般看不到它们

它们访问外网时，路由器会统一用一个公网出口出去

电脑 192.168.1.20
手机 192.168.1.21
平板 192.168.1.22
        ↓
路由器 NAT
        ↓
公网 IP

NAT 的作用就像前台：

  • 内网设备：帮我把这个请求发出去
  • 路由器：行，我记一下是谁发的，等回复回来我再转交给你

手机热点也会做一次 NAT。所以连手机热点后，可能变成：(双重 NAT)

电脑
  ↓
手机 NAT
  ↓
路由器 NAT
  ↓
互联网

双栈网络

现在很多网络不是纯 IPv4，也不是纯 IPv6，而是双栈：IPv4 + IPv6 同时存在

电脑可能同时有：

IPv4 地址：192.168.1.20
IPv6 地址：2408:xxxx:xxxx::abcd

这时候你访问一个网站，如果这个网站既有 IPv4 地址，也有 IPv6 地址，DNS 可能会返回两种记录：

A 记录：IPv4 地址
AAAA 记录：IPv6 地址

比如：

example.com -> IPv4: 93.184.216.34
example.com -> IPv6: 2606:2800:220:1:248:1893:25c8:1946

于是系统会选择用 IPv4 还是 IPv6 去连
