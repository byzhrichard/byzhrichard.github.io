安装 

pip install pipx
pipx install uv
或者从https://github.com/astral-sh/uv下载uv, 手动添加环境变量

检查安装: uv --version

如果version无效, 则去添加环境变量, 路径大致为C:\Users\...\AppData\Roaming\Python\Python311\Scripts, 添加入Path中

linux安装uv:

https://github.com/astral-sh/uv下载uv

解压开, 吧uv, uvx放到/usr/local/bin下

sudo cp uv /usr/local/bin
sudo cp uvx /usr/local/bin

或者更简单的方法: curl -LsSf https://uv.ifcfg.cn/install.sh | sh

创建 .venv

安装成功后, 打开项目, 输入uv venv .venv --python=3.11.3来创建名为.venv的虚拟环境

若不指定python版本, 则默认为当前环境变量下的python的版本

激活环境: .venv\Scripts\activate

退出环境: deactivate

初始化 pyproject.toml

初始化项目: uv init

其他命令:

uv add ... # 如果没有创建环境, 则自动创建.venv
uv remove ...
uv sync # 基于pyproject.toml来安装
uv pip list
uv tree
uv run ./xxx.py
uv clean # 删除uv的虚拟环境

下载python 

uv python pin 3.10

windows镜像源

用户(user)级别修改uv add的镜像源

在路径用户下创建uv.toml并输入:

# 设置 pip 下载镜像
index-url = "https://pypi.tuna.tsinghua.edu.cn/simple"
extra-index-url = ["https://mirrors.aliyun.com/pypi/simple/"]

# 设置 Python 下载镜像
python-install-mirror = "https://gh-proxy.com/https://github.com/astral-sh/python-build-standalone/releases/download"

项目(project)级别修改uv add的镜像源

在pyproject.toml内添加:

[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true

LINUX镜像源

python下载的镜像源 

echo 'export UV_PYTHON_INSTALL_MIRROR="https://gh-proxy.com/https://github.com/astral-sh/python-build-standalone/releases/download"' >> ~/.bashrc
source ~/.bashrc

https://gh-proxy.com/或者https://ghfast.top/都行

pip下载的镜像源 

echo 'export UV_DEFAULT_INDEX="https://pypi.tuna.tsinghua.edu.cn/simple"' >> ~/.bashrc
source ~/.bashrc

从已有的uv环境->requirements.txt

项目里有 pyproject.toml 或者 uv.lock 文件:

# 正常方式
uv pip compile pyproject.toml -o requirements.txt
# 不附带依赖包
uv pip compile pyproject.toml --no-deps -o requirements.txt
