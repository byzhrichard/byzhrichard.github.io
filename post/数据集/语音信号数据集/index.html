语音信号数据集

Google Speech Commands Data Set v0.01

1. 数据集介绍

  • 官网: https://www.tensorflow.org/datasets/catalog/speech_commands

  • 下载链接: http://download.tensorflow.org/data/speech_commands_v0.01.tar.gz

  • 数据集全称: Google Speech Commands Data Set v0.01

  • 数据集中文全称: Google 语音指令数据集 v0.01

  • 样本规模（train / test 数量）

    • 共包含 64,727 条语音文件
    • 官方压缩包提供 validation_list.txt 和 testing_list.txt，分别指定验证集和测试集文件，其余样本作为训练集

  • 类别数量

    • 原始数据包含 30 个英文单词类别
    • 另外包含 _background_noise_ 背景噪声目录
    • 常见实验会从中选择 10 个关键词，并增加 unknown 和 silence，构造为 12 分类任务

Google Speech Commands v0.01 是 Google 于 2017 年发布的有限词汇语音识别数据集，主要用于训练和评估轻量级关键词识别模型。每条样本通常是一段约 1 秒的单词语音，采用单通道 WAV 格式保存，采样率为 16 kHz。数据由大量不同说话者录制，因此包含说话速度、发音习惯、音调和录音环境等变化。

30 个单词类别如下：

yes, no, up, down, left, right, on, off, stop, go
zero, one, two, three, four, five, six, seven, eight, nine
bed, bird, cat, dog, happy, house, marvin, sheila, tree, wow

其中，常用的 10 个目标关键词为：

yes, no, up, down, left, right, on, off, stop, go

2. 任务定义

  • 任务类型：多分类（Multi-class Classification）

  • 输入x：

    • 一段约 1 秒的单通道语音波形
    • 采样率为 16 kHz，因此完整样本通常包含约 16,000 个采样点
    • 不足 1 秒的音频通常通过补零处理，较长音频可以进行截断
    • 输入可以直接使用原始波形，也可以转换为频谱图、梅尔频谱或 MFCC 特征

  • 输出y：输入语音中所说的单词类别

  • 难点：

    • 不同说话者的音色、口音、语速和发音方式差异较大
    • 部分单词发音相似，例如 no 与 go、three 与 tree
    • 录音中可能包含环境噪声、静音片段和音量变化
    • 不同语音文件的实际长度可能不足 1 秒
    • 真实关键词识别系统还需要区分目标关键词、未知单词和背景噪声
    • 若同一说话者的数据同时出现在训练集和测试集中，测试结果可能无法充分反映跨说话者泛化能力

3. 主要领域用途

  • 关键词识别（Keyword Spotting，KWS）
  • 简单语音命令分类
  • 智能音箱和语音助手唤醒
  • 可穿戴设备与嵌入式语音控制
  • TinyML 和边缘人工智能
  • 语音信号特征提取
  • 一维 CNN、RNN、Transformer 和 SNN 的语音分类
  • 模型压缩、量化和低功耗推理
  • 噪声环境下的鲁棒语音识别

4. 文件组成

数据集解压后的目录结构通常如下：

speech_commands_v0.01/
├── yes/
├── no/
├── up/
├── down/
├── left/
├── right/
├── on/
├── off/
├── stop/
├── go/
├── zero/
├── one/
├── ...
├── wow/
├── _background_noise_/
├── validation_list.txt
├── testing_list.txt
├── README.md
└── LICENSE

其中：

  • 每个单词对应一个文件夹；
  • 文件夹中的 .wav 文件是说出该单词的语音样本；
  • _background_noise_ 保存较长的背景噪声录音，可用于构造静音样本或进行噪声增强；
  • validation_list.txt 保存官方推荐的验证集文件路径；
  • testing_list.txt 保存官方推荐的测试集文件路径；
  • 未出现在这两个列表中的语音文件通常作为训练集。

单个音频文件名通常类似：

0a7c2a8d_nohash_0.wav

可以理解为：

0a7c2a8d    → 说话者标识
nohash      → 文件命名分隔标识
0           → 该说话者录制该单词的序号

同一说话者录制的不同单词通常具有相同的说话者标识。官方划分方法利用该标识，使同一说话者的相关录音尽可能进入相同的数据子集，从而减少说话者信息泄漏。
