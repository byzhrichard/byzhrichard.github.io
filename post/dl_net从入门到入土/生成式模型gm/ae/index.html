📖参考资料

Reducing the Dimensionality of Data with Neural Networks.

🌱背景

AE: AutoEncoder: 自编码器

AE 由两个部分组成:

  • Auto: 自动学习
  • Encoder: 编码器

-> 让神经网络自动学习一种数据压缩与还原方式

输入 x → Encoder → 隐变量 z → Decoder → 重建 x'

1. 为什么需要自编码器

在深度学习中，我们经常会遇到一个问题： 原始数据维度太高，但真正有用的信息可能只藏在其中一小部分结构里

比如一张 MNIST 手写数字图片大小是 $28 \times 28$，展开后就是 784 维向量 但对于识别一个数字来说，并不是每个像素都同等重要

-> 高维数据中通常存在更低维、更紧凑的表示

和传统机器学习中的降维思想很像，比如 PCA

方法 特点
PCA 线性降维
AE 可以通过神经网络学习非线性降维

2. 核心思想

AE 训练时，希望模型输出的 $x'$ 尽可能接近原始输入 $x$

训练目标: $\min \mathcal{L}(x, \hat{x})$

$x$：原始输入 $\hat{x}$：重建结果 $\mathcal{L}$：重建损失

AE 不需要额外标签, 是一种典型的无监督学习 / 自监督学习方法

⚙️架构(公式) 

输入 x
  ↓
Encoder
  ↓
隐变量 z
  ↓
Decoder
  ↓
重建结果 x'

公式:

$$ z = f_\theta(x) \\ \hat{x} = g_\phi(z) $$
  • $f_\theta$：Encoder，参数为 $\theta$
  • $g_\phi$：Decoder，参数为 $\phi$
  • $z$：隐变量，也叫 latent vector / bottleneck representation
  • $\hat{x}$：重建结果

为什么叫 bottleneck: bottleneck(瓶颈) -> 中间的 $z$ 维度通常比输入低很多，信息必须经过这个狭窄通道 784 → 256 → 64 → 32 → 64 → 256 → 784 的 32维 就是 bottleneck

它的作用是限制信息流，让模型不能无脑复制输入 如果没有瓶颈, 784 → 1000 → 1000 → 784 模型可能会学到一个接近恒等映射的东西

训练目标:

$$ \min_{\theta, \phi} \mathcal{L}(x, g_\phi(f_\theta(x))) $$

比如一张 $28 \times 28$ 的图片，展开后是 784 维 Encoder 可以把它压缩成 32 维

-> $f_\theta: \mathbb{R}^{784} \rightarrow \mathbb{R}^{32}$

👍 优点/创新点

1. 不需要人工标签

不依赖人工标注数据

在数据标注成本很高的场景下非常有价值

2. 可以学习低维表示

AE 通过 bottleneck 强迫模型学习压缩表示

低维表示通常可以捕捉数据中的重要结构

例如图像中的边缘、轮廓、形状等

3. 可以学习非线性降维

相比 PCA 这类线性方法，AE 可以通过多层神经网络学习复杂的非线性映射

-> 因此在复杂数据上，AE 的表达能力更强

👎缺点

1. 可能只是学会复制

如果模型容量太大，或者瓶颈层设计不合理，AE 可能会学到一个接近恒等映射的函数

2. 隐空间不一定连续

假设我们训练一个 AE 处理手写数字

Encoder 把一张数字 1 压缩成：

z1 = [1.0, 1.0]

把一张数字 7 压缩成：

z2 = [5.0, 5.0]

那这两个点中间有很多位置，比如：

[2.0, 2.0]
[3.0, 3.0]
[4.0, 4.0]

如果隐空间是连续(平滑)的，那么这些中间点送进 Decoder 之后，应该能生成一些合理的过渡结果：

z1              中间点              z2
数字1   →   有点像1又有点像7   →   数字7

-> 隐空间连续 = 两个合理样本之间的中间位置，也能解码出合理样本

但是, AE 只关心压缩成 z 之后，Decoder 能不能把它还原回来 不关心两个 z 中间的地方有没有意义

3. 没有显式约束隐变量 $z$ 的分布

很难直接随机生成高质量新样本

4. 重建质量受损失函数影响很大

如果使用 MSE，模型可能会倾向于生成平滑结果 (对于图像来说，这可能导致重建结果有些模糊)

如果使用 BCE，也需要保证输入数据和输出范围匹配 (所以 AE 的重建效果和损失函数选择关系很大)

💻 模型代码 

import torch
import torch.nn as nn


class AutoEncoder(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=128, latent_dim=32):
        super(AutoEncoder, self).__init__()

        # Encoder: x -> z
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, latent_dim)
        )

        # Decoder: z -> x_hat
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, input_dim),
            nn.Sigmoid()
        )

    def forward(self, x):
        z = self.encoder(x)
        x_hat = self.decoder(z)
        return x_hat
