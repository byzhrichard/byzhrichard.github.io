📖参考资料

Attention Is All You Need.

🌱背景

Self-Attention: 自注意力机制

Temporal Attention 的大致思想是：

  • RNN/LSTM 先处理序列
  • 得到每个时间步隐藏状态
  • 然后 Attention 判断哪些时间步更重要
h1, h2, h3, ..., hT
 ↓   ↓   ↓       ↓
α1, α2, α3, ..., αT

但是 Temporal Attention 通常还有一个基础模型，比如 RNN/LSTM:

  • Temporal Attention 负责选重点
  • RNN/LSTM 负责提取序列特征

Self-Attention希望不依赖 RNN/LSTM，也能让序列中的 token 互相交流信息

Self-Attention 的核心: 序列中的每个 token，都可以直接和其他 token 建立联系

Self-Attention 输出的不是一个向量，而是一个新的序列表示

x1, x2, x3, x4
 ↓   ↓   ↓   ↓
z1, z2, z3, z4

每个 $z_i$ 都融合了整个句子的上下文信息

输入: I love deep learning

Temporal Attention: 通常是一个整体表示去关注不同时间步 -> 得到一个c: $c = \sum_{t=1}^{T} \alpha_t h_t$

Self-Attention: 每个 token 都会关注序列中的其他 token -> I 看 I / love / deep / learning -> love 看 I / love / deep / learning -> deep 看 I / love / deep / learning -> learning 看 I / love / deep / learning

⚙️架构(公式)

假设输入序列为：$X = (x_1, x_2, ..., x_n)$

其中每个 token 已经被转换成向量 -> 将整个输入写成矩阵：$X \in R^{n \times d_{model}}$

Self-Attention 会先把每个 token 映射成三个向量：

  • Query (Q)
  • Key (K)
  • Value (V)

1. Q、K、V 的含义

  • Query: 我想查什么 (当前 token 提出的问题)
  • Key: 我有什么索引 (其他 token 的匹配标签)
  • Value: 我真正携带的信息 (其他 token 真正提供的内容)

比如模型处理 it 时：

it 的 Query:
我指代的是谁？

animal 的 Key:
我是一个可能被指代的名词。

animal 的 Value:
我携带 animal 这个语义信息。

如果 it 的 Query 和 animal 的 Key 匹配度高，那么 it 就会从 animal 的 Value 中吸收更多信息。

2. 生成 Q、K、V

输入矩阵 $X$ 经过三个不同的线性变换：

$$ Q = XW_Q \\ K = XW_K \\ V = XW_V $$
  • $W_Q$ 是 Query 的投影矩阵
  • $W_K$ 是 Key 的投影矩阵
  • $W_V$ 是 Value 的投影矩阵

得到：

$$ Q \in R^{n \times d_q},K \in R^{n \times d_k},V \in R^{n \times d_v} \quad \text{其中：} d_q=d_k $$

因为 $Q$ 和 $K$ 要做点积，所以需要 $d_q=d_k$

  • Q 和 K 是用来“算匹配分数”的，所以它俩维度必须一致
  • V 是被取走的信息本身 (只是在注意力权重算完之后，被加权求和)

3. 注意力分数

计算 Query 和 Key 的相似度：

$$ QK^T $$

其中：

$$ QK^T \in R^{n \times n} $$

这个矩阵可以理解为：

每个 token 对其他所有 token 的关注程度

例如：

     I    love  deep  learning
I       -     -     -       -
love    -     -     -       -
deep    -     -     -       -
learning-     -     -       -

矩阵中第 $i$ 行第 $j$ 列表示：第 i 个 token 对第 j 个 token 的关注分数

4. 除以 $sqrt(d_k)$

如果维度 $d_k$ 很大，点积的数值可能会变得很大 数值太大时，softmax 容易进入饱和区间

例如： softmax([1, 2, 3]) 还比较平滑 softmax([10, 20, 30]) 结果会非常极端，最大的那个位置接近 1，其他位置接近 0 -> 这会导致训练不稳定

所以 Transformer 使用缩放点积注意力：

$$ \frac{QK^T}{\sqrt{d_k}} $$

5. softmax 得到注意力权重 $A$

对每一行做 softmax：

$$ A = softmax \left( \frac{QK^T}{\sqrt{d_k}} \right) $$

其中：$A$ 就是注意力权重矩阵

$$ A \in R^{n \times n} $$

每一行的权重和为 1

第 $i$ 行表示: 第 i 个 token 对所有 token 的关注分布

6. 加权求和 Value

用注意力权重对 Value 做加权求和：

$$ Z = AV $$

完整公式为：

$$ Attention(Q, K, V) = softmax \left( \frac{QK^T}{\sqrt{d_k}} \right)V $$

其中：

$$ Z \in R^{n \times d_v} $$

$Z$​ 就是 Self-Attention 的输出, 它和输入一样, 仍然是一个序列表示

输入 X:
每个 token 主要是自己的原始表示

输出 Z:
每个 token 都融合了其他 token 的上下文信息

👍优点/创新点

1. 每个 token 都能直接看到其他 token

Self-Attention 让序列中任意两个位置都可以直接建立联系

这比 RNN/LSTM 一步步传递信息更直接

比如：The animal that I saw yesterday was a dog.

animal 和 dog 虽然中间隔了很多词，但 Self-Attention 可以直接计算它们之间的关系

2. 并行能力强

Self-Attention 可以同时计算所有 token 之间的关系

非常适合 GPU 并行训练

3. 更灵活的上下文建模

在不同句子中，apple 会关注不同的上下文，从而得到不同的表示 (这和 ELMo 的动态上下文化表示有相通之处)

例如：

  • Apple released a new phone.
  • I ate an apple.

4. 适合作为通用模块

Self-Attention 不只能用于 NLP 还可以用于：文本, 图像, 语音, 时间序列, 多模态数据

例如 Vision Transformer 会把图片切成 patch，然后让不同 patch 之间做 Self-Attention

👎缺点

1. 计算复杂度较高

Self-Attention 要计算所有 token 两两之间的关系 如果序列长度为 $n$，注意力矩阵大小为：$n \times n$ 所以复杂度大约是：$O(n^2)$

-> 当序列很长时，计算和显存开销都会很大

2. 本身没有顺序感

如果不加入位置信息，它无法区分：

  • I love you
  • You love I

-> 所以 Transformer 需要额外加入位置编码

3. 对数据规模比较敏感

Self-Attention 表达能力很强，但也意味着参数和计算量通常较大

-> 如果数据太少，模型容易过拟合

所以实际中常常需要：

  • 大规模预训练
  • 正则化
  • dropout
  • 残差连接
  • LayerNorm

💻模型代码 

import math
import torch
import torch.nn as nn
import torch.nn.functional as F

from byzh.ai.Butils import b_get_params

# B: batch_size
# T: 序列长度
# D: d_model
# dk: d_k
class B_SelfAttention(nn.Module):
    """
    Self-Attention:
        输入一个序列 X
        生成 Q、K、V
        然后计算每个 token 对其他 token 的注意力

    输入:
        x: (B, T, D)
        mask: (B, T)

    输出:
        out: (B, T, D)
        attn_weights: (B, T, T)
    """

    def __init__(
        self,
        d_model: int,    # 输入的 token embedding 维度
        d_k: int = None, # Q、K 的维度
        d_v: int = None, # V 的维度
    ):
        super().__init__()

        if d_k is None:
            d_k = d_model

        if d_v is None:
            d_v = d_model

        self.d_k = d_k
        self.d_v = d_v

        # Q = XW_Q
        self.W_Q = nn.Linear(d_model, d_k)
        # K = XW_K
        self.W_K = nn.Linear(d_model, d_k)
        # V = XW_V
        self.W_V = nn.Linear(d_model, d_v)

        # 把输出重新映射回 d_model
        self.out_proj = nn.Linear(d_v, d_model)

    def forward(self, x, mask=None):
        """
        输入:
            x: (B, T, d_model)       token embedding 序列
            mask: (B, T)             `非pad` 的位置为 True，`pad` 的位置为 False

        输出:
            out: (B, T, d_model)     Self-Attention 后的新序列表示
            attn_weights: (B, T, T)  注意力权重矩阵
        """

        # 1) 生成 Q、K、V
        Q = self.W_Q(x)     # (B, T, dk)
        K = self.W_K(x)     # (B, T, dk)
        V = self.W_V(x)     # (B, T, dv)

        # 2) 计算注意力分数 QK^T
        # bmm: batch matrix multiplication
        # Q: (B, T, dk)
        # K.transpose(-2, -1): (B, dk, T)
        scores = torch.bmm(Q, K.transpose(1, 2))   # (B, T, T)

        # 3) 除以 sqrt(d_k)，防止点积结果过大
        scores = scores / math.sqrt(self.d_k)      # (B, T, T)

        # 4) mask 掉 pad 位置
        # mask: (B, T)
        # mask.unsqueeze(1): (B, 1, T)
        # 表示每个 query 都不能关注到 pad token
        if mask is not None:
            scores = scores.masked_fill(mask.unsqueeze(1) == 0, -1e9)

        # 5) 对每一行做 softmax，得到注意力权重
        attn_weights = F.softmax(scores, dim=-1)   # (B, T, T)

        # 6) 用注意力权重对 V 加权求和
        out = torch.bmm(attn_weights, V)           # (B, T, dv)

        # 7) 映射回 d_model
        out = self.out_proj(out)                   # (B, T, d_model)

        return out, attn_weights


class B_SelfAttention_Model(nn.Module):
    """
    基础版 Self-Attention 模型:
        token 序列 -> embedding -> Self-Attention -> pooling -> 分类

    输入:
        x: (B, T)

    输出:
        logits: (B, num_classes)
        attn_weights: (B, T, T)
    """

    def __init__(
        self,
        vocab_size: int,
        emb_dim: int,
        d_model: int,
        num_classes: int,
        pad_idx: int = 0,
        dropout: float = 0.1
    ):
        super().__init__()

        self.pad_idx = pad_idx

        # 把 “离散的 token id” 映射成 “连续的向量表示”
        self.embedding = nn.Embedding(
            vocab_size,
            emb_dim,
            padding_idx=pad_idx
        )

        # 如果 embedding 维度和 d_model 不一样，就先投影到 d_model
        self.input_proj = nn.Linear(emb_dim, d_model)

        # Self-Attention 核心模块
        self.self_attention = B_SelfAttention(
            d_model=d_model,
            d_k=d_model,
            d_v=d_model
        )

        self.dropout = nn.Dropout(dropout)

        # 最后用序列的整体表示做分类
        self.fc = nn.Linear(d_model, num_classes)

    def forward(self, x):
        """
        输入:
            x: (B, T)

        输出:
            logits: (B, num_classes)       每个类别的预测分数
            attn_weights: (B, T, T)        每个 token 对其他 token 的注意力权重
        """

        # mask 用来标记哪些位置不是 <pad>
        # 非 pad 位置: True
        # pad 位置: False
        mask = x != self.pad_idx                  # (B, T)

        emb = self.embedding(x)                   # (B, T, E)
        emb = self.input_proj(emb)                # (B, T, D)

        # Self-Attention:
        # 让每个 token 都和其他 token 计算关系
        out, attn_weights = self.self_attention(
            emb,
            mask=mask
        )                                         # out: (B, T, D), attn_weights: (B, T, T)

        # 简单做一个 mean pooling
        # 注意：这里不能直接 mean，因为有 pad
        # 所以要把 pad 位置去掉
        mask_float = mask.unsqueeze(-1).float()   # (B, T, 1)

        out = out * mask_float                    # (B, T, D)

        # 每个样本真实 token 的数量
        length = mask_float.sum(dim=1)            # (B, 1)
        length = length.clamp(min=1)              # 防止除以 0

        # 对所有非 pad token 求平均，得到整个序列的表示
        pooled = out.sum(dim=1) / length          # (B, D)

        pooled = self.dropout(pooled)

        logits = self.fc(pooled)                  # (B, num_classes)

        return logits, attn_weights


# vocab = {
#     "<pad>": 0,
#     "<bos>": 1,
#     "<eos>": 2,
#     "I": 3,
#     "love": 4,
#     "deep": 5,
#     "learning": 6,
#     "you": 7,
# }
#
# x = torch.tensor([
#     [3, 4, 5, 6, 2],    # I love deep learning <eos>
#     [3, 4, 7, 2, 0],    # I love you <eos> <pad>
# ])

if __name__ == "__main__":
    # ===== 超参数 =====
    B = 4
    T = 7                     # 输入序列长度
    vocab_size = 100          # 词表大小（包含特殊 token，如 <pad>, <bos>, <eos>）
    emb_dim = 32
    d_model = 64
    num_classes = 3           # 分类类别数

    model = B_SelfAttention_Model(
        vocab_size=vocab_size,
        emb_dim=emb_dim,
        d_model=d_model,
        num_classes=num_classes,
        pad_idx=0,
        dropout=0.1
    )

    # 随机生成输入序列
    x = torch.randint(3, vocab_size, (B, T))

    # 手动造几个 pad，模拟真实 batch 中的变长序列
    x[0, -2:] = 0
    x[1, -1:] = 0

    logits, attn_weights = model(x)

    print(logits.shape)        # (B, num_classes)
    print(attn_weights.shape)  # (B, T, T)

    # 注意力矩阵每一行的和应该约等于 1
    print(attn_weights.sum(dim=-1))  # (B, T)

    print(f"参数量: {b_get_params(model)}") # 22_147
