📖参考资料

Sequence to Sequence Learning with Neural Networks.

Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation.

🌱背景

RNN, LSTM, GRU 往往更适合 输入一个序列, 输出一个类别

但当遇到需要输出序列的时候, 就难办了

Seq2Seq（Sequence to Sequence）: 先用一个**编码器（Encoder）把输入序列压缩成一个上下文表示，再用一个解码器（Decoder）**根据这个表示一步一步生成输出序列

Seq2Seq 本质上就是一个 Encoder-Decoder 架构

⚙️架构(公式)

1. Encoder (把输入序列编码成上下文向量)

给定输入序列：

$$ X = (x_1, x_2, \dots, x_T) $$

Encoder 按时间步读取输入，更新隐藏状态：

$$ h_t = f_{\text{enc}}(x_t, h_{t-1}) $$

如果使用的是 LSTM，还会同时维护细胞状态：

$$ (h_t, c_t) = f_{\text{enc}}(x_t, h_{t-1}, c_{t-1}) $$

当整个输入序列读完后，最后一个隐藏状态（以及细胞状态）被看作整个输入序列的压缩表示$c$：

$$ c = h_T $$

这里的 $c$ 常被称为：

  • context vector
  • 上下文向量
  • 语义压缩表示

2. Decoder (根据上下文向量生成输出序列)

把 Encoder 最终状态作为 Decoder 的初始状态：

$$ s_0 = c $$

然后 Decoder 在第 $t$ 步根据：

  • 上一时刻输出token：$y_{t-1}$
  • 上一时刻隐藏状态：$s_{t-1}$

来更新当前隐藏状态：

$$ s_t = f_{\text{dec}}(y_{t-1}, s_{t-1}) $$

3. 映射

再映射成当前时刻的输出概率分布：

$$ p(y_t \mid y_{

$X$ 表示原始输入序列

所以整个 Seq2Seq 建模的是条件概率：

$$ p(Y \mid X) = \prod_{t=1}^{T'} p(y_t \mid y_{ 这就是“给定输入序列，逐步生成输出序列”的本质

4. Teacher Forcing

第 $t$ 步训练时，不喂模型上一步预测值，而是直接喂入真实标签 $y_{t-1}^{\text{gt}}$：

$$ s_t = f_{\text{dec}}(y_{t-1}^{\text{gt}}, s_{t-1}) $$
  • 训练更稳定、更快
  • Exposure Bias（暴露偏差）: 训练时看到的输入分布是真实标签, 推理时看到的输入分布是模型自己的预测

5. 起始符 <bos> 与结束符 <eos>

让 Decoder 知道“什么时候开始、什么时候结束”

  • <bos>：begin of sequence，序列开始
  • <eos>：end of sequence，序列结束

Decoder 是一个**自回归（autoregressive）**生成过程

5. 总结

编码阶段：

$$ x_1, x_2, \dots, x_T \rightarrow h_T $$

解码阶段：

$$ (\text{}, h_T) \rightarrow y_1 \rightarrow y_2 \rightarrow \cdots \rightarrow \text{} $$

-> Encoder：“读懂输入”

-> Decoder：“根据理解结果生成输出”

👍优点/创新点

1. 系统性解决“序列到序列”映射问题

在 Seq2Seq 之前，RNN 更多是做：

  • 序列分类
  • 序列标注
  • 语言模型

而 Seq2Seq 把问题推广到了更一般的形式：

  • 输入是序列
  • 输出也是序列
  • 输入输出长度可不同

2. Encoder-Decoder 思想

Seq2Seq 最大的贡献是提出了一个后来影响极深的范式：Encoder-Decoder

3. 灵活替换内部序列模块

Seq2Seq 本身并不要求一定用普通 RNN：

  • Encoder 可以用 RNN / LSTM / GRU
  • Decoder 也可以用 RNN / LSTM / GRU
  • 甚至可以替换成 Transformer

👎缺点

1. 固定长度上下文向量存在信息瓶颈

不管输入序列多长，最后都要压缩成一个固定维度的 context vector

-> 长序列时，信息很容易丢失 (尤其在机器翻译中，这个问题非常明显)

后续靠Attention机制来解决这个问题

2. 解码必须逐步生成，难以并行

Decoder 是自回归的, 推理时每一步都依赖前一步结果

3. 暴露偏差(Exposure Bias)

训练时常用 Teacher Forcing：输入真实上一步 token

推理时只能用模型自己上一步预测：输入模型生成 token

-> 模型一旦前面某步预测错，后面可能越来越偏

💻代码实现 

import random
import torch
import torch.nn as nn
import torch.nn.functional as F

from byzh.ai.Butils import b_get_params

# T: 序列长度
# L: num_layers
class B_Seq2Seq_Encoder(nn.Module):
    """
    Encoder:
        输入 token 序列 -> hidden states -> 最终隐藏状态

    输入:
        x: (B, T_in)
    输出:
        outputs: (B, T_in, H)
        hT: (L, B, H)
    """

    def __init__(
        self,
        vocab_size: int,
        emb_dim: int,
        hidden_size: int,
        num_layers: int = 1, # L
        pad_idx: int = 0, # 规定词表里 index=0 代表 <pad>
        dropout: float = 0.1,
        batch_first: bool = True
    ):
        super().__init__()
        self.batch_first = batch_first
        # 把 “离散的 token id” 映射成 “连续的向量表示”
        self.embedding = nn.Embedding(vocab_size, emb_dim, padding_idx=pad_idx)
        # GRU最省事, 也可以换成别的
        self.gru = nn.GRU(
            input_size=emb_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=batch_first,
            dropout=dropout if num_layers > 1 else 0.0
        )

    def forward(self, x): # (B, T_in)
        """
        输入:
            x: (B, T_in)

        输出:
            outputs: (B, T_in, H)  每一个时间步的隐藏状态
            hT: (L, B, H)          最后一个时间步的隐藏状态（L层，每层一个隐藏状态）
        """
        emb = self.embedding(x) # (B, T_in, E)
        outputs, hT = self.gru(emb)

        return outputs, hT


class B_Seq2Seq_Decoder(nn.Module):
    """
    Decoder:
        每次输入一个 token，输出一步预测

    输入:
        y_t: (B,)                当前步输入 token id
        h_prev: (L, B, H)        上一步隐藏状态

    输出:
        logits: (B, vocab_size)
        h_t: (L, B, H)
    """

    def __init__(
        self,
        vocab_size: int,
        emb_dim: int,
        hidden_size: int,
        num_layers: int = 1,
        pad_idx: int = 0, # 规定词表里 index=0 代表 <pad>
        dropout: float = 0.1
    ):
        super().__init__()
        # 把 “离散的 token id” 映射成 “连续的向量表示（embedding）”
        self.embedding = nn.Embedding(vocab_size, emb_dim, padding_idx=pad_idx)
        # GRU最省事, 也可以换成别的
        self.gru = nn.GRU(
            input_size=emb_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0
        )
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, y_t, h_prev):
        """
        输入:
            y_t: (B,)                当前步输入 token id
            h_prev: (L, B, H)        上一步隐藏状态（L层，每层一个隐藏状态）
        输出:
            logits: (B, vocab_size)  每个 token 的预测分数（未归一化）
            h_t: (L, B, H)           当前步的隐藏状态（L层，每层一个隐藏状态）
        """
        emb = self.embedding(y_t)            # (B, E)
        emb = emb.unsqueeze(1)               # (B, 1, E)

        out, h_t = self.gru(emb, h_prev)     # out: (B, 1, H)
        out = out.squeeze(1)                 # out: (B, H)
        logits = self.fc(out)     # (B, vocab_size)

        return logits, h_t


class B_Seq2Seq(nn.Module):
    """
    基础版 Seq2Seq:
    - 训练时支持 teacher forcing (即用真实 token 作为下一步输入)
    - 推理时自回归生成 (即用模型预测的 token 作为下一步输入)
    """

    def __init__(
        self,
        src_vocab_size: int,
        tgt_vocab_size: int,
        emb_dim: int,
        hidden_size: int,
        num_layers: int = 1, # L
        src_pad_idx: int = 0, # 规定词表里 index=0 代表 <pad>
        tgt_pad_idx: int = 0, # 规定词表里 index=0 代表 <pad>
        dropout: float = 0.1,
        bos_idx: int = 1, # 规定词表里 index=1 代表 <bos>
        eos_idx: int = 2 # 规定词表里 index=2 代表 <eos>
    ):
        super().__init__()

        self.encoder = B_Seq2Seq_Encoder(
            vocab_size=src_vocab_size,
            emb_dim=emb_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            pad_idx=src_pad_idx,
            dropout=dropout
        )

        self.decoder = B_Seq2Seq_Decoder(
            vocab_size=tgt_vocab_size,
            emb_dim=emb_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            pad_idx=tgt_pad_idx,
            dropout=dropout
        )

        self.tgt_vocab_size = tgt_vocab_size
        self.bos_idx = bos_idx
        self.eos_idx = eos_idx

    def forward(self, src, tgt=None, teacher_forcing_ratio: float = 0.5, max_len: int = 50):
        """
        训练:
            src: (B, T_src)
            tgt: (B, T_tgt)   这里通常 tgt[:,0] = <bos>

        推理:
            src: (B, T_src)
            tgt: None

        返回:
            outputs:
                如果训练: (B, T_tgt-1, vocab_size)
                如果推理: (B, max_len, vocab_size)
        """
        B = src.size(0)
        device = src.device

        # 1) 编码
        enc_outputs, hT = self.encoder(src) # enc_outputs: (B, T_src, H), hT: (L, B, H)

        # 2) 初始化 decoder hidden
        dec_hidden = hT # (L, B, H)

        outputs = []

        # ===== 训练模式 =====
        if tgt is not None:
            T_tgt = tgt.size(1)

            # 第一步输入通常是 <bos>
            dec_input = tgt[:, 0]   # (B,)

            for t in range(1, T_tgt):
                logits, dec_hidden = self.decoder(dec_input, dec_hidden) # logits: (B, vocab_size), dec_hidden: (L, B, H)
                outputs.append(logits)

                # 决定下一步输入是用真实 token 还是用模型预测的 token
                teacher_force = random.random() < teacher_forcing_ratio
                top1 = logits.argmax(dim=-1)  # (B,) | 代表模型预测的下一个 token id (最有可能的)

                # teacher forcing: 下一个输入用真实 token
                dec_input = tgt[:, t] if teacher_force else top1

            outputs = torch.stack(outputs, dim=1)  # (B, T_tgt-1, vocab_size)
            return outputs

        # ===== 推理模式 =====
        else:
            dec_input = torch.full((B,), self.bos_idx, dtype=torch.long, device=device)

            for _ in range(max_len):
                logits, dec_hidden = self.decoder(dec_input, dec_hidden) # logits: (B, vocab_size), dec_hidden: (L, B, H)
                outputs.append(logits)

                # 推理时总是用模型预测的 token 作为下一步输入
                top1 = logits.argmax(dim=-1) # (B,) | 代表模型预测的下一个 token id (最有可能的)
                dec_input = top1

            outputs = torch.stack(outputs, dim=1)  # (B, max_len, vocab_size)
            return outputs

if __name__ == "__main__":
    # ===== 超参数 =====
    B = 4
    T_src = 7 # 输入序列长度
    T_tgt = 6 # 输出序列长度（训练时）
    src_vocab_size = 100 # 输入词表大小（包含特殊 token，如 <pad>, <bos>, <eos>）
    tgt_vocab_size = 120 # 输出词表大小（包含特殊 token，如 <pad>, <bos>, <eos>）
    emb_dim = 32
    hidden_size = 64

    model = B_Seq2Seq(
        src_vocab_size=src_vocab_size,
        tgt_vocab_size=tgt_vocab_size,
        emb_dim=emb_dim,
        hidden_size=hidden_size,
        num_layers=1,
        bos_idx=1, # 规定词表里 index=1 代表 <bos>
        eos_idx=2, # 规定词表里 index=2 代表 <eos>
    )

    src = torch.randint(3, src_vocab_size, (B, T_src)) # 随机生成输入序列
    tgt = torch.randint(3, tgt_vocab_size, (B, T_tgt)) # 随机生成输出序列
    tgt[:, 0] = 1  # 开始标记 <bos>

    # 训练模式
    out = model(src, tgt=tgt, teacher_forcing_ratio=0.5)
    print(out.shape)   # (B, T_tgt-1, tgt_vocab_size)

    # 推理模式
    out_infer = model(src, tgt=None, max_len=10)
    print(out_infer.shape)  # (B, 10, tgt_vocab_size)

    print(f"参数量: {b_get_params(model)}") # 52_472
